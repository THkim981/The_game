# Bolt Rush Prototype

React + TypeScript + Vite 기반의 방치형/도박 하이브리드 프로토타입입니다. 기본 수익, 안전 업그레이드, Heat/Luck/Insight, 3단계 리스크 실험 UI가 포함되어 있습니다.

## 스크립트

- 개발 서버: `npm run dev`
- 린트: `npm run lint`
- 포맷: `npm run format` / 체크만 `npm run format:check`
- 테스트: `npm run test` (Vitest + jsdom)
- 빌드: `npm run build`

## 구조
- `src/App.tsx`: 자원 루프, 업그레이드, 리스크 실험 로직 및 UI
- `src/test/setup.ts`: 테스트 환경 설정 (`@testing-library/jest-dom`)
- `vite.config.ts`: React 플러그인 + Vitest 설정

## 추후 확장 아이디어
- Insight로 메타 업그레이드 구매하는 화면 분리
- Heat/Cash 오프라인 계산 로직 정교화
- 리스크 확률/보상 테이블을 서버 혹은 JSON으로 분리해 밸런싱 편의성 확보

## 현재 밸런스 스냅샷 (프로토타입)
- 기본 수익: 14 C/s (BASE_INCOME)
- Tick 주기: 0.2s (리소스 갱신), Cash/Chips/Heat는 per-second 값을 tick에 맞춰 나눠서 적산
- Heat 충전: 기본 100/180초 ≈ 0.556 Heat/s, 배터리 레벨당 +10% 가속
- Chips 생성: 기본 1G/30s ≈ 0.0333 G/s, 정제 레벨당 +5% 가속
 시작 자원: Cash 420, Chips 16, Heat 0, Luck 0, Insight 0, Prestige 0

 금고 확장: 시작 200C, 성장 1.14, 효과 +5%p 곱
- 프린터: 시작 60C, 성장 1.12, 효과 +10% 가산
 숫자 단위 표기 규칙
- 자동 수금: 시작 100C, 성장 1.14, 효과 +5%p (최대 20레벨)

### Prestige (초기 도입)
 - Prestige 자원: Shard, 시작 0
 - 보너스: 수익 배율에 `1 + 0.05 * log10(1 + Prestige)` 곱
 - 획득량: `floor(max(0, log10(1 + (Cash + Chips*1000 + Insight*10000)) - 8) ^ 1.4)`
 - 리셋 효과: Cash/Chips/Heat/Luck/Insight/업그레이드/버프/permBoost 초기화, Prestige 누적 유지
- 이자 계약: 시작 260C, 성장 1.18, 효과 R = 0.1*log10(1 + Cash/1000)*(1 + 0.07*Lv)

### 도박(실험) 리스크 테이블
- 낮음: 10G, 확률 {대성공 5%, 성공 60%, 실패 25%, 대실패 10%}, 부스트 1.35~1.8x/10분, 쿨다운 60s
- 중간: 25G, 확률 {8%, 57%, 22%, 13%}, 부스트 1.7~2.6x/10분, 쿨다운 120s
- 극단: 60G, 확률 {15%, 50%, 20%, 15%}, 부스트 2.4~5x/10분, 대성공 시 영구 +2% (누적 상한 20%), 쿨다운 180s
- 초극단: 150G, 확률 {18%, 45%, 20%, 17%}, 부스트 3.2~7.5x/12분, 대성공 시 영구 +3%, 쿨다운 300s
- Luck 보정: L(0~100)일 때 대성공 확률×(1+0.3·L/100), 성공 확률×(1+0.6·L/100); 남는 확률은 실패/대실패 비율(5:2, 22:13, 20:15, 20:17)로 분배, 무효는 고정 0%

### UI/UX 피드백 장치
- 숫자: AnimatedNumber로 부드럽게 보간 (0.24s), Heat 게이지 폭 트랜지션 0.18s
- 도박 결과: 토스트 2.6s 노출, 톤별(성공/경고/실패) 라디얼 FX 0.9s

### 숫자 단위 표기 규칙
- 사용 순서: K(10E3) → M(10E6) → B(10E9) → T(10E12) → Qa(10E15) → Qi(10E18) → Sx(10E21) → Sp(10E24) → Oc(10E27) → Nn(10E30)

### 초반 진행 의도
- 약 2~3분 내 Heat 100 도달, 낮은 리스크 1~2회 시도 가능
- 시작 현금/칩 여유로 프린터/자동 수금/정제 초기 레벨을 바로 구매 가능하도록 조정
